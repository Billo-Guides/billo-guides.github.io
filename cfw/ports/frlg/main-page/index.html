FireRed & LeafGreen RNG (CFW)

A guide for RNG manipulating Pokémon in the Switch ports of FireRed & LeafGreen using GRASS, a tool developed by LegoFigure11.
Introduction

In FireRed & LeafGreen, RNG manipulation lets you control the outcomes of in-game events, including the internal details of Pokémon you encounter. Generation 3 RNG mechanics have been studied extensively over the years, and a lot of that same knowledge can be applied to the Switch versions of these games.

This guide explains the existing research while also highlighting any differences or quirks specific to RNG manipulation on the Switch ports. It's designed to be useful whether you're new to RNG abuse in Generation 3 or are already familiar with the concepts.

Required Software
Tools and Applications
  • GRASS by LegoFigure11
Installation Instructions
  1. Download the latest GRASS release and extract it to an empty folder in a known location on your computer.
  2. Copy the bundled atmosphere folder to the root of your Switch SD card, then reboot the console into your chosen CFW environment.

The included atmosphere folder contains a custom fork of sys-botbase, which allows GRASS to communicate with your console over the same network. The bundled sys-botbase fork adds extra features, most notably USB support, which can provide a more stable connection if your Wi-Fi connection is unreliable.

USB support is not enabled by default and must be manually configured, this requires some extra initial setup that is not covered in this guide. If you wish to enable USB support, visit this page and come back once setup is complete.

GRASS Setup

When opening GRASS for the first time you will be presented with something similar to the above image. Before doing anything else you will need to confirm that you can connect to the program with no issues by doing the following:

  1. Launch FireRed/LeafGreen and press the A button to dismiss the title screen, you should now be on the main menu.
  2. Type the IP address of your Switch console into the Switch IP: field of GRASS.
    • Your Switch IP address can be found by launching the Homebrew Menu then pressing the Y button (port is not relevant).
    • If you are using a USB connection, ensure that your USB cable is properly connected to the console instead.
  3. Click the Connect button on GRASS. If connection was sucessful you should now see that the Seed, TID and SID fields have updated automatically.

Great! - You should now have GRASS installed to your computer and have sys-botbase functioning properly on your console. Double check that you have a successful connection to GRASS before proceeding as this will not be covered later.

This section covers common community terminology used throughout RNG manipulation. It's intended to give you a clear understanding of key terms and phrases, helping you avoid confusion when following the guide.

Common Terminology Table
Term Definition
Initial Seed When launching the game you are given a "random" hexadecimal value that ultimately determines the outcome of anything random that happens in-game.
Advances Otherwise known as "frames", this term is used to describe how many times the RNG seed has progressed since we initially launched the game.
Delay Otherwise known as "offset", this term is used to describe the number of advances that are between the final button input and the advance/frame we actually end up hitting.
Method The way the game generates a Pokémon. Different generation "methods" result in different outcomes for the target Pokémon, most notably the IVs the Pokémon is generated with. Most of the time, wild Pokémon are generated using Method H1. Gifts, Static Encounters, and Roamers typically use Method 1.
VBlank Otherwise known as "vertical blanking", is the function that is used to advance the games RNG, and is responsible for clearing the screen to draw the next frame. This function can affect the method a Pokémon is generated with.
PID Otherwise known as "Personality ID", is a unique identifier for each Pokémon. This value is closely tied to the details of the Pokémon, such as its IVs, nature, gender, ability, and most importantly shiny status.
TID/SID Otherwise known as "Trainer ID/Secret ID", is a unique identifier for each trainer. These values are used to determine if a Pokémon is shiny based on that Pokémon's PID. We can manipulate our TID/SID to make a specific PID result in that Pokémon being shiny (covered later!).

PID, TID & SID Relationship

As mentioned in the table, your TID & SID determine whether a Pokémon will appear shiny. When a Pokémon is generated, the game compares its PID against your Trainer ID and Secret ID. If the values match the required condition, the Pokémon is generated as shiny. In theory, any PID can be shiny with the correct TID/SID combination.

TID can be controlled through bruteforce (botting), while SID is controlled by RNG manipulation (frame perfect button input). Any value between 00000-65535 is possible for both the TID and SID. Manipulating both the TID and SID allows complete control over the spreads that will be shiny in the save file, however, you can also bruteforce a specific TID for aesthetic/vanity purposes and avoid SID manipulation entirely if you wish.

Consider manipulating both the TID and the SID if your goal is to obtain a specific shiny spread or a rare combination of IVs and nature. This allows you to target a specific PID that will result in the desired spread (e.g, 6 perfect IVs) and manipulate a TID/SID combination that will result in the Pokémon generating as shiny.

Note - Most common spreads do not require a specific TID/SID to be shiny. For example, if you are only aiming for a general competitive spread (such as low Attack or Speed IVs), TID/SID manipulation is not necessary. However, lucking into highly specific combinations like a 6IV shiny by pure chance is astronomically unlikely, so TID/SID manipulation is basically a requirement for those targets.
