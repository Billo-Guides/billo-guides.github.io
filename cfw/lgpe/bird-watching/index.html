Legendary Bird Hunting (LGPE)

This guide provides coverage on how to Moarencounterbots, a branch of a SysBot.NET fork by Lusamine. This program is used to automate the process of hunting overworld legendary birds (Articuno, Zapdos & Moltres) in Let's Go! Pikachu & Eevee.

From here it is assumed you are using a CFW Switch console running Atmosphere. There is no alternative for this type of RNG abuse at this point in time.

Getting Started
You will need:
  • Moarencounterbots
  • .NET 8.0
  • Sys-botbase
Installation instructions:
  • Download .NET 8.0 SDK and run the installation file until complete. This is a dependency for Moarencounterbots.
  • Download the latest sys-botbase release and extract the atmosphere folder to the root of your Switch SD card.
  • Reboot the console into SysCFW.
Moarencounterbots does not come with pre-compiled releases, this means you are expected to build the application from source code using Visual Studio. If you are unfamiliar with how to do this, click the button below for a tutorial.
Click here if the Pokemon you wish to RNG abuse is tied to Spawner RNG!

Moarencounterbots is a fork of SysBot.NET that provides extra encounterbots that cannot be found in the base program, including EncBotBirdWatchLGPE. This encounterbot checks the RNG state to determine when an overworld legendary bird will spawn and will continually enter Mt. Moon PokeCénter until it finds a close enough target.

It's possible to configure the bot to look for a specific bird target OR any of the three, it's also possible to hunt for only Shiny targets.

From here this guide assumes you are using a CFW Switch console running Atmosphere.

Getting Started:

You will need:
  • SysBot.NET Moarencounterbots - Latest commit
  • sys-botbase - Latest Release
For building the program:
  • .NET 4.7 SDK
  • Visual Studio 2022
Unsure why you need extra components?
  • sys-botbase is a Switch system module which allows the console to communicate with your computer, SysBot.NET is unable to connect to your console without sys-botbase properly installed.
  • .NET 4.7 SDK is required to build the application, Visual Studio is the application that will allow you to build the application.

You will need to compile the source code as this program does not provide a pre-compiled release. Read through the documentation found on the Official Sysbot.NET Wiki for more information on how to compile the application using Visual Studio & sys-botbase installation. For further instructions consider visiting my other guide which covers how to compile this application in some extra detail.

SysBot Tab Overview

This portion of the guide covers all the information you need to know about each of the tabs in order to run EncBotBirdWatchLGPE.

The default configuration is for Scarlet/Violet, you will first need to change the current game settings to use Let's Go encounterbots. Upon launching the program for the first time and closing it, a file named config.json will appear, right click and open it in any text editor and change the Mode to 100.

After changing the mode, then running SysBot.exe again you will be presented with a screen similar to the one shown below in Figure 1.

Figure 2 - Hub tab overview


Hub tab Overview

The Hub tab is where we decide the type of encounter we are doing and which Pokémon to hunt for. Scroll down to EncounterLGPE and set your conditions, click on each of the fields and you will be met with a short description about them at the bottom of the window.

Now scroll down to StopConditions and set your filters for your legendary bird of choice, you can be as specific with the filters as you like, such as filtering for specific IVs or shiny type, or have the bot continue the routine if the legendary bird is not shiny.

Suggested settings and a detailed overview of each setting can be found on the Moarencounterbots Wiki, it's recommended to read this page in its entirety before starting, it contains a lot of useful information!

Figure 3 - Logs tab overview


Logs tab Overview

The Log tab is where we monitor what is going on in-game, it shows us what the bot is doing in real-time and will contain all the information you need to know, such as when a legendary bird has spawned in the overworld, or if it's a shiny variant.

Figure 4 - Bots tab overview


Bots tab Overview

The Bots tab is where we add bots, select the EncBotBirdWatch option in the dropdown menu, type your switch IP into the field and click Add to add a new instance of EncBotBirdWatch.

With setup out of the way, it's time hunt some legendary birds!

Let's Go to Work!

The goal is to spawn the legendary bird of choice in the overworld. we will use the bot to predict when a bird will spawn and to advance towards our target. It will then output the result and pause the game using the home button so that we can catch it!

Notes and tips:
  • If you do not want the bot to stop on non-shiny results set the StopOnAllBirdsLGPE setting to FALSE in StopConditions
  • If you wish to hunt for a specific bird you must set the StopOnSpecies setting to your bird of choice in StopConditions
  • The bot assumes you are using the Pokécenter at Mt. Moon and have defeated/caught the legendary bird trio in their original static locations.
  • The legendary birds are sky spawns, meaning you must have unlocked the ability to encounter sky spawns in-game, given shortly after defeating the Elite 4.

Shiny Hunting Strategy

To get optimal chances of finding a shiny legendary bird you should capture every bird that spawns of your chosen species. Upon capturing a Pokemon your shiny odds will increase for the next Pokemon that spawns of the same species.

For this reason it's recommended to have StopOnAllBirdsLGPE set to TRUE, you can then chain capture your chosen species for boosted shiny odds.

In this example we will be looking for any legendary bird at Mt. Moon Pokécenter, you can be more specific with the filters if you wish, such as filtering for a specific or shiny bird.
